# Setup AI Agents — Пошаговая инструкция

> Этот файл объясняет **как настроить агентов под твой проект**. Один раз сделать, потом пользоваться.

---

## Часть 1: Почему именно так

### Твоя ошибка прошлый раз

В предыдущем проекте у тебя были "агенты" в `agents/`:
```python
# agent_backend.py
def create_database_models(self) -> str:
    result = self.ask_claude(prompt)  # ← вызвал Claude
    global_context.set_backend_result(...)  # ← сохранил текст в JSON
    return result  # ← вернул текст
```

**Это не агент.** Это автоматизированный prompt-runner. Он не пишет файлы, не правит код, не понимает контекст. Просто генерирует текст один раз.

### Что такое настоящий агент

Настоящий агент:
1. ✅ **Читает** файлы из проекта
2. ✅ **Пишет** новые файлы
3. ✅ **Редактирует** существующие
4. ✅ **Запускает** команды (тесты, миграции)
5. ✅ **Помнит** контекст между задачами
6. ✅ **Откатывается** если что-то сломал

Примеры таких агентов: **Claude Code**, **Cursor**, **Aider**, **Cline**, **Continue.dev**.

### Что нужно сделать чтобы они работали "под твой проект"

Их нужно **сконфигурировать** под проект. Это делается через специальные файлы:
- `CLAUDE.md` — Claude Code читает его автоматически
- `.cursorrules` — Cursor читает
- `.aider.conf.yml` — Aider читает

В этих файлах живут **правила, контекст, ограничения** для агента.

## Часть 5: Как работать с агентом

### Правильный workflow (одна задача = один чат)

```bash
# 1. Открой Claude Code
claude

# 2. Дай задачу из ROADMAP (скопируй промпт целиком)
> [вставляешь промпт Этапа 0]

# 3. Агент работает, задаёт вопросы
# 4. Ты подтверждаешь или корректируешь
# 5. Агент закончил → проверь критерии приёмки

# 6. Закрой сессию
> /exit

# 7. Сделай git commit (если агент не сделал)
git status
git diff
git commit -am "feat: complete stage 0 - project setup"

# 8. Следующая задача — НОВЫЙ чат
claude
> [промпт Этапа 1]
```

### Почему "один чат = одна задача"

- Контекст не загромождается
- Если что-то пошло не так — легко откатить
- Каждая задача атомарна в git
- Можно остановиться и продолжить позже

### Чего НЕ делать

❌ **Не давай агенту "сделай всё MVP за раз"** — он зальётся, контекст переполнится, начнёт нести чушь.

❌ **Не правь код руками в середине задачи агента** — он не увидит изменения и перезапишет.

❌ **Не запускай несколько сессий параллельно в одном проекте** — конфликты.

❌ **Не делай git commit за агента в середине работы** — может сбить.

✅ **Что делать:**
- Дал задачу → дождался ответа → проверил → commit → следующая
- Если задача большая — попроси разбить
- Если непонятно — спрашивай агента "почему ты так сделал?"

## Часть 6: Полезные команды Claude Code

```bash
# Запуск в текущей папке
claude

# Запуск с задачей сразу
claude "fix the bug in availability calculator"

# Разовая команда
claude -p "show me git status"

# Continue последнюю сессию
claude --continue

# Список сессий
claude --resume

# Поменять модель (если есть доступ)
claude --model claude-opus-4-7-20250214
```

### Внутри сессии

```
/clear          — очистить контекст (новая задача)
/cost           — сколько стоит сессия
/help           — список команд
/permissions    — настройка прав на файлы/команды
/exit           — выйти
```

## Часть 7: Если что-то идёт не так

### "Агент не понимает что делать"

Проверь:
1. CLAUDE.md в корне проекта?
2. В первом сообщении сессии написано "прочитай CLAUDE.md"?
3. ROADMAP.md существует и обновлён?

Если да — переформулируй задачу, добавь больше деталей.

### "Агент сделал не то"

```
> /clear
> отмени последние изменения через git и попробуй заново.
  на этот раз делай X а не Y.
```

Или если совсем плохо:
```bash
git reset --hard HEAD~1  # откатить последний commit
git clean -fd            # удалить новые файлы
```

### "Агент застрял в цикле ошибок"

Если агент 3+ раза пытается исправить одну и ту же ошибку:
1. `/clear`
2. Дай агенту контекст: вывод ошибки + что пытались
3. Спроси: "напиши план как это решить, ничего пока не делая"
4. Утверди план → дай зелёный свет

### "Контекст переполнен, агент тупит"

```
/clear  # начать чистый чат
# дай ему только нужный контекст:
# - текущая задача из ROADMAP
# - 1-2 релевантных файла
```

## Часть 8: Многоагентность (когда дорастёшь)

**Сейчас тебе НЕ нужно** запускать несколько агентов. Один Claude Code = всё MVP.

Когда тебе понадобится несколько:
- Большой проект (50+ файлов меняются регулярно)
- Команда людей
- Параллельные фичи

Тогда можешь:
1. Разные `git worktree` для разных фич, в каждом свой Claude Code
2. Или использовать **Cursor Composer** (несколько изменений одновременно)
3. Или **Aider** для batch tasks

Но для MVP клиники — **один агент это твой компанион**, и больше пока не нужно.

## Часть 9: Стоимость

### Claude Code на Pro плане ($20/мес)

- Включено: фиксированная квота сообщений
- Хватит ли на MVP? **Скорее всего да**, особенно если:
  - Чёткие промпты (как в ROADMAP)
  - `/clear` между задачами
  - Не "делай всё подряд"

### Claude Code через API

- Платишь за токены
- Sonnet 4.6: ~$3 input / $15 output на 1M токенов
- На MVP при правильном использовании: **$30-100 за весь проект**

### Что влияет на стоимость

| Действие | Стоимость |
|----------|-----------|
| Одна простая задача (1 файл) | $0.10-0.50 |
| Этап ROADMAP (10-15 файлов) | $1-3 |
| Refactor большого модуля | $2-5 |
| "Сделай мне весь backend" (плохой промпт) | $20-50 |

**Совет:** мониторь `/cost` в начале. Если задача обходится >$5 — что-то не так с подходом.

## Часть 12: Когда обращаться ко мне (в этом чате)

Используй меня для:
- ✅ **Стратегические решения** ("стоит ли добавить feature X?")
- ✅ **Архитектурные споры** ("как лучше моделировать Y?")
- ✅ **Ревью того что сделал агент** ("вот код, проверь")
- ✅ **Если агент совсем не понимает** — приходи с контекстом, я разберу
- ✅ **Обновление CLAUDE.md / ROADMAP** — когда новые требования

НЕ используй меня для:
- ❌ "Напиши мне функцию X" — это работа агента
- ❌ "Создай файл Y" — то же
- ❌ Мелкие правки кода — у меня нет доступа к твоему диску

**Правило большого пальца:**
- Думать, планировать, ревьюить → я (этот чат)
- Писать код, править файлы, делать commits → Claude Code в твоей системе
